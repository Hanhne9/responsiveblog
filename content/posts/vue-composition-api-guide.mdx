# Vue 3 Composition API: Complete Developer Guide

The Composition API is one of the most significant additions to Vue 3, offering a more flexible and powerful way to organize component logic.

## Why Composition API?

The Composition API addresses several limitations of the Options API:

- Better TypeScript support
- Improved code reusability
- More flexible component organization
- Better logic composition

## Basic Setup

 # {{ title }}


 Count: {{ count }}


 Increment
 



import { ref, computed } from 'vue'

const title = ref('My Component')
const count = ref(0)

const increment = () => {
 count.value++
}

const doubleCount = computed(() => count.value * 2)

## Reactive References

\`\`\`javascript
import { ref, reactive } from 'vue'

// Primitive values
const count = ref(0)
const message = ref('Hello')

// Objects
const user = reactive({
 name: 'John',
 age: 30,
 email: 'john@example.com'
})

// Accessing values
console.log(count.value) // 0
console.log(user.name) // 'John'

## Computed Properties

\`\`\`javascript
import { ref, computed } from 'vue'

const firstName = ref('John')
const lastName = ref('Doe')

const fullName = computed(() => {
 return `${firstName.value} ${lastName.value}`
})

// Writable computed
const fullNameWritable = computed({
 get() {
 return `${firstName.value} ${lastName.value}`
 },
 set(value) {
 [firstName.value, lastName.value] = value.split(' ')
 }
})
\`\`\`

## Watchers

\`\`\`javascript
import { ref, watch, watchEffect } from 'vue'

const count = ref(0)
const message = ref('Hello')

// Watch single source
watch(count, (newValue, oldValue) => {
 console.log(`Count changed from ${oldValue} to ${newValue}`)
})

// Watch multiple sources
watch([count, message], ([newCount, newMessage], [oldCount, oldMessage]) => {
 console.log('Multiple values changed')
})

// Watch effect
watchEffect(() => {
 console.log(`Count is ${count.value}`)
})
\`\`\`

## Custom Composables

\`\`\`javascript
// composables/useCounter.js
import { ref } from 'vue'

export function useCounter(initialValue = 0) {
 const count = ref(initialValue)

 const increment = () => count.value++
 const decrement = () => count.value--
 const reset = () => count.value = initialValue

 return {
 count,
 increment,
 decrement,
 reset
 }
}

// In component
import { useCounter } from './composables/useCounter'

export default {
 setup() {
 const { count, increment, decrement, reset } = useCounter(10)

 return {
 count,
 increment,
 decrement,
 reset
 }
 }
}
\`\`\`

## Provide/Inject

\`\`\`javascript
// Parent component
import { provide, ref } from 'vue'

export default {
 setup() {
 const theme = ref('dark')
 
 provide('theme', theme)
 
 return { theme }
 }
}

// Child component
import { inject } from 'vue'

export default {
 setup() {
 const theme = inject('theme', 'light') // default value
 
 return { theme }
 }
}

## Migration from Options API

\`\`\`javascript
// Options API
export default {
 data() {
 return {
 count: 0,
 message: 'Hello'
 }
 },
 computed: {
 doubleCount() {
 return this.count * 2
 }
 },
 methods: {
 increment() {
 this.count++
 }
 },
 mounted() {
 console.log('Component mounted')
 }
}

// Composition API equivalent
import { ref, computed, onMounted } from 'vue'

export default {
 setup() {
 const count = ref(0)
 const message = ref('Hello')
 
 const doubleCount = computed(() => count.value * 2)
 
 const increment = () => {
 count.value++
 }
 
 onMounted(() => {
 console.log('Component mounted')
 })
 
 return {
 count,
 message,
 doubleCount,
 increment
 }
 }
}
\`\`\`

## Best Practices

1. **Use script setup** for cleaner syntax
2. **Group related logic** together
3. **Extract reusable logic** into composables
4. **Use TypeScript** for better development experience
5. **Prefer reactive() for objects** and ref() for primitives

## Conclusion

The Composition API provides a more powerful and flexible way to build Vue components. It's especially beneficial for complex components and when you need to share logic between components.
